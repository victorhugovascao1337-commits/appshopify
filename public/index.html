      help Como funciona e onde instala cada coisa expand_more

      São duas peças em duas lojas diferentes. Não confunda:

      storefront

      1. O script vai na VITRINE

      É a loja de onde o cliente chega pelo anúncio. O botão “Instalar na vitrine” aqui embaixo coloca um ScriptTag nela — ele detecta o produto e redireciona. Não mexe no tema. Precisa do escopo write_script_tags na vitrine.

      shopping_cart_checkout

      2. A Storefront API vai na loja de CHECKOUT

      Só é preciso para o Checkout direto (carrinho pronto). É lá que o painel monta o carrinho e gera o link de pagamento. Ele usa um token da Storefront API da loja de checkout. Jeito mais fácil, que funciona em qualquer loja e plano:

      • Canal Headless (recomendado): na loja de checkout, instale o app Headless pela App Store → Storefronts → API access → copie o Public access token e cole no campo “Token Storefront” que aparece no aviso “Destino do cliente” abaixo. Funciona mesmo sem custom app.
      • Custom App (se a loja tiver “Desenvolver apps”): marque os escopos unauthenticated_* em Storefront API — o painel gera o token sozinho.

      Enquanto não colar/liberar, o cliente cai na página do produto da loja de checkout — nada quebra. O aviso “Destino do cliente” abaixo mostra se já está pronta.

      Dica: teste sem redirecionar abrindo a vitrine com ?noredirect=1 no fim da URL.

      🔗 Antes de começar — 3 pontos que confundem

      Crie o app dentro do admin da própria loja (Apps → Develop apps) — funciona em qualquer loja, sem depender de organização.

      Domínio = o endereço .myshopify.com da loja (ex.: minhaloja.myshopify.com) — não o seu e-mail nem o domínio personalizado.

      O token é o Admin API access token gerado ao instalar o app (API credentials → Install app). Não é a API key nem o secret.

      🔑 Escopos do Admin API que você precisa marcar
      • read_orders Obrigatório Vendas, faturamento, ticket e a contagem que faz a rotação pular de loja.
      • read_products Obrigatório Catálogo, AutoMatch por SKU e o mapeamento vitrine → checkout.
      • write_script_tags read_script_tags Opcional Instalar e remover o script de redirect na vitrine, direto pelo painel — sem mexer no tema.
      • write_orders Opcional Criar o pedido do pós-compra digital automaticamente.
      • read_themes write_themes Opcional Alternativa ao script tag: injetar no theme.liquid. Só se preferir mexer no tema.
      • unauthenticated_read_product_listings unauthenticated_read_product_tags Storefront API Ler produtos publicados pela vitrine sem login do cliente. Fica na seção Storefront API do app, não no Admin API.
      • unauthenticated_write_checkouts unauthenticated_read_checkouts Storefront API Criar/ler checkout na loja de destino (levar o carrinho pronto). Também é Storefront API.

      Os unauthenticated_* ficam em Storefront API (outra seção do app), não em Escopos do Admin API.
      Marcou escopo depois de instalar? Salve, clique em Instalar app de novo e gere um token novo — o antigo não ganha as permissões.

      ▸ Como obter o token (1 minuto)
      1. No admin da loja: Configurações → Apps e canais de vendas → Desenvolver apps (se for a primeira vez, clique em "Permitir desenvolvimento de apps personalizados").
      2. Criar um app → dê um nome qualquer (ex.: "Painel").
      3. Em Configuração → Escopos do Admin API, marque os escopos da lista acima e salve.
      4. Vá em Credenciais da API → Instalar app.
      5. Copie o token de acesso do Admin API (começa com shpat_ — ele só aparece uma vez!) e cole aqui.
      ⚠ Deu "This app can't be installed on this store"?

      Isso é a distribuição do app, não o OAuth. Em Partner Dashboard → seu app → Distribution:

      • Custom distribution → só instala em uma loja/organização escolhida. É o erro mais comum.

      • Public distribution → instala em qualquer loja (precisa de review da Shopify só para listar na App Store; para uso próprio, o link de instalação já funciona).

      A escolha da distribuição é definitiva depois de selecionada — se errou, crie outro app.

      🔑 Escopos que o app precisa pedir

      No Partner App você não marca no admin da loja: declare os escopos no app (Partner Dashboard → App setup → Admin API access scopes, ou scopes no shopify.app.toml). Eles vão no parâmetro scope da URL de autorização e aparecem na tela de permissões que o lojista aceita.

      • read_orders Obrigatório Vendas, faturamento, ticket e a contagem que faz a rotação pular de loja.
      • read_products Obrigatório Catálogo, AutoMatch por SKU e o mapeamento vitrine → checkout.
      • write_script_tags read_script_tags Opcional Instalar e remover o script de redirect na vitrine, direto pelo painel — sem mexer no tema.
      • write_orders Opcional Criar o pedido do pós-compra digital automaticamente.
      • read_themes write_themes Opcional Alternativa ao script tag: injetar no theme.liquid. Só se preferir mexer no tema.
      • unauthenticated_read_product_listings unauthenticated_read_product_tags Storefront API Ler produtos publicados pela vitrine sem login do cliente. Fica na seção Storefront API do app, não no Admin API.
      • unauthenticated_write_checkouts unauthenticated_read_checkouts Storefront API Criar/ler checkout na loja de destino (levar o carrinho pronto). Também é Storefront API.

      O painel já pede todos esses no OAuth. Os unauthenticated_* exigem a Storefront API habilitada no app.
      Mudou os escopos com o app já instalado? O lojista precisa reautorizar (rode o OAuth de novo) — o token antigo mantém só as permissões antigas.
